# SOP4：订单取消处理

**核心目标：** 核实订单号并根据当前订单状态给出准确回复。

## 操作步骤

1. **索要订单号**
   - 检查：客户提出取消请求时，是否已提供具体订单号？
   - 动作：未提供时，先不要做任何假设，直接回复：“请问您的订单号是多少？”（拿到订单号后再进行下一步）。

2. **查询订单状态**
   - 动作：在系统后台输入订单号，查看该订单目前的实际处理进度。

3. **根据状态精准回复（严格按以下分类处理）**
   - **情况 A：订单显示【Unpaid】**
     - 话术：“您可以直接在您的账户中取消订单。”
     - 纪律要求：此情况属于客户自助操作范围，绝对不要为此类工单创建人工服务或转接其他客服。

   - **情况 B：订单显示【Paid】/【Awaiting】/【Processing】**
     - 话术：“请告知我们取消订单的原因，你的专属业务员会为您处理。”
     - 动作：发送完上述话术收集原因后，立即将对话转交给人工客服跟进处理后续事宜。

   - **情况 C：订单显示【Shipped】**
     - 话术：“订单已发货，无法取消。”
