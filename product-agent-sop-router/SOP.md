### SOP_5：样品申请

# 当前任务：处理用户询问如何申请样品、希望先买样品测试的请求

## 场景描述

* 用户询问如何申请样品，或表示想先购买样品测试。
* 示例：
* I’d like to order a sample of this SKU.
* I need alot of samples to start this business.

## 执行步骤 (严格按顺序)

**Step 1: 检查用户是否提供具体产品信息（满足任一项即可）**

* 可识别产品信息清单（任一项命中即视为已提供）：
* SKU
* 产品名称
* 产品链接

**Step 2: 按信息完整度分支处理**

* IF 仅提供产品类型/模糊描述（未提供 SKU、产品名称、产品链接）：
* 动作：
1. 使用“回复模板3”引导用户补充具体信息。
2. **【必须】调用 `need-human-help-tool1`工具。**

* ELSE 已提供具体产品信息：
* 动作：调用 `query-product-information-tool1`（Product API）查询价格、产品链接和 MOQ。

**Step 3: 按 Product API 查询结果分支处理**

* IF Product API 查询无结果：
* 动作：表明未找到相关信息，请检查后重试。

* IF 查询成功且 MOQ = 1：
* 动作：使用“回复模板1”告知可直接下单，并提供价格和产品链接。

* IF 查询成功且 MOQ > 1：
* 动作：
1. 使用“回复模板2”告知起订量与价格区间，并说明可提交样品申请。
2. **【必须】调用 `need-human-help-tool1`工具。**

## 回复模板

* 回复模板1：有 SKU + MOQ = 1
[SKU] 支持单件购买，当前价格：[价格]
您可直接点击链接下单：[产品链接]

* 回复模板2：有 SKU + MOQ > 1
[SKU] 起订量为 [MOQ] 件，价格为：[价格区间]
您所需数量少于起订量，可提交样品申请。您的专属客户经理会协助您处理，请邮件至{业务员邮箱}(session_metadata.sale email)咨询。

* 回复模板3：仅提供产品类型/模糊描述
您希望申请 [用户描述的产品类型] 的样品。
为了更好地为您处理，请提供以下信息：
具体产品（SKU/产品链接/产品名称）
需要几件样品
个人使用还是商业使用
您的联系方式
信息补充完整后，您的专属客户经理会协助您处理，请邮件至{业务员邮箱}(session_metadata.sale email)咨询。

* 限制：【严格遵守】回复语种必须和 `Target Language`一致。
