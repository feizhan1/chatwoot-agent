# SOP Router List 维护规则

## 技能触发条件
- 用户明确要求“根据 `sop.md` 更新路由器里的可选 SOP 列表”。
- 用户要求“对齐 `sop.md` 与 `system-prompt.md` 的 SOP 摘要/路由目标”。
- 用户在新增、删除或改写某个 SOP 后，要求同步更新路由清单。
- 用户提到“可选的 SOP 列表”“Available SOP List”“路由目标”这类区块需要重写或校正。

## 输入前提
- 必填输入 1：源文件 `sop.md` 路径（示例：`order-agent-sop-router/sop.md`）。
- 必填输入 2：目标路由文件 `system-prompt.md` 路径（示例：`order-agent-sop-router/system-prompt.md`）。
- 选填输入：英文路由文件 `system-prompt.en.md` 路径；若存在，应同步更新语义一致的英文 SOP 列表。
- 源文件必须包含可识别的 SOP 分段标题（如 `### SOP_1：...`）。
- 目标文件必须存在“可选的 SOP 列表”区块（或英文同义区块）。

## 执行步骤
1. 读取 `sop.md`，按 SOP 分段提取：
   - SOP 编号（`SOP_1`、`SOP_2`...）
   - 分段标题（冒号后的短标题）
   - `当前任务` 句（若存在，作为首要触发描述来源）
   - 触发场景关键词（如“取消订单”“退款/退货”“物流进度”“以图搜图”）
2. 生成路由摘要行，格式固定为：
   - 中文：`* **SOP_X**: <触发条件句>`
   - 英文：`* **SOP_X**: <Trigger sentence>.`
   - 中文示例：`* **SOP_1**: 当用户咨询订单问题且上下文未识别到有效订单/追踪号时触发。`
3. 用新摘要覆盖 `system-prompt.md` 中“可选的 SOP 列表”区块内容。
4. 若存在 `system-prompt.en.md`，同步生成等价英文摘要并覆盖对应区块。
5. 做一致性校验并输出结果。

## 摘要编写规则
- 每个 SOP 只保留一行，按编号升序排列。
- 摘要只写“触发条件句”，推荐中文以“当/若”起句，英文以 “Triggered when ...” 起句。
- 禁止保留短名称（如“缺失订单号处理”）和“——”解释结构。
- 禁止写执行细节、工具调用、流程限制、转人工动作等非触发条件信息。
- 目标是让模型只根据一句话判断“什么场景下返回该 SOP”。
- 禁止照搬执行话术、长段 IF/ELSE、工具参数细节。
- 禁止新增不存在的 SOP 编号，也不能遗漏 `sop.md` 里的编号。
- 若 `sop.md` 信息不完整，优先保守表达触发场景，不臆造业务规则。

## 区块定位规则
- 中文区块起始标记优先匹配：`## 可选的 SOP 列表`
- 英文区块起始标记优先匹配：`## Available SOP List`
- 区块结束位置：下一个 `## ` 二级标题之前。
- 仅替换该区块，不修改其它章节。

## 交付与校验
- 交付时应明确：
  - 修改了哪些文件
  - 共更新了多少条 SOP 摘要
  - 是否完成中英文一致性同步
- 最低校验项：
  - SOP 编号集合与 `sop.md` 完全一致
  - 无重复编号、无遗漏编号
  - 摘要语义与 `sop.md` 触发场景一致
