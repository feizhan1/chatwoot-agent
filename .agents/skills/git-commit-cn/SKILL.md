# Git 中文提交技能

## 技能触发条件
- 用户要求执行 `git commit`、提交当前改动、代为提交代码。
- 用户要求提交信息必须是中文，或要求提交说明更易读。
- 用户希望在 commit 正文中看到“变更摘要”和“改动文件路径”。

## 输入前提
- 运行目录必须位于 Git 仓库内。
- 允许读取 `git status` 与 `git diff` 结果生成提交说明。
- 若用户明确指定提交范围或提交信息，优先遵循用户要求。

## 执行步骤
1. 检查提交前状态。
- 运行 `git status --short`。
- 若无改动，直接返回“当前无可提交内容”，不执行 commit。

2. 确定提交范围。
- 若用户给出文件/目录路径，仅暂存这些路径。
- 若用户未指定范围：
  - 若已有 staged 内容，保持现状。
  - 若没有 staged 内容，执行 `git add -A`。

3. 生成中文提交说明。
- 提交说明格式：
  - 标题：`<type>: <中文总结>`
  - 正文分两段：
    - `变更摘要：` 1-3 条中文要点（每条一句）
    - `变更文件：` 文件路径列表（每行一个路径）
- 正文模板：
  - `变更摘要：`
  - `- 摘要 1`
  - `- 摘要 2`
  - ``
  - `变更文件：`
  - `- path/a`
  - `- path/b`
- `type` 优先使用 `feat`、`fix`、`docs`、`refactor`、`chore` 中最贴切的一类。
- 中文总结要求：
  - 8-30 字，聚焦“本次改动做了什么”。
  - 禁止纯英文整句。
  - 禁止空泛描述（如“更新一下”“改了点东西”）。
- 变更摘要要求：
  - 基于 `git diff --cached` 生成，不得臆造。
  - 只写结果，不写过程。
  - 单条建议 12-40 字。
- 文件路径要求：
  - 从已暂存文件中生成（使用 `git diff --cached --name-only`）。
  - 使用仓库相对路径。
  - 默认完整列出全部路径；若超过 20 个文件，可按目录聚合并附代表路径。

4. 执行提交。
- 运行带标题+正文的提交命令，例如：
  - `git commit -m "<type>: <中文总结>" -m "变更摘要：\n- 摘要1\n- 摘要2\n\n变更文件：\n- path/a\n- path/b"`
- 默认不使用 `--no-verify`，除非用户明确要求跳过钩子。

5. 提交完成后合并当前分支到 `main`。
- 记录提交前当前分支：`git branch --show-current`。
- 若当前分支是 `main`，跳过合并并标记“已在 main，无需合并”。
- 若当前分支不是 `main`：
  - 先执行 `git checkout main`。
  - 再执行 `git merge --ff-only <原分支名>`。
- 若 `main` 不存在或合并失败，明确返回失败原因与下一步建议。

6. 执行 `origin` 推送。
- 在 `main` 分支执行：`git push origin main`。
- 默认不修改 `origin` 的 `pushurl` 配置。
- 若 `origin` 推送失败，明确返回失败原因与下一步建议。

7. 执行 `github` 推送（默认执行）。
- 优先使用已存在的 `github` 远程：`git push github main`。
- 若不存在 `github` 远程：
  - 直接返回失败并提示先配置：`git remote add github https://github.com/feizhan1/chatwoot-agent.git`
  - 不自动新增远程配置。

8. 提交与推送后校验。
- 运行 `git show --name-only --pretty=format:"%H%n%s" -1` 输出提交哈希与标题。
- 运行 `git remote -v` 输出远程信息（至少包含 `origin`，若执行过可包含 `github`）。
- 再次运行 `git status --short`。
- 若钩子产生新的未提交改动，明确告知用户并询问是否继续追加提交。

## 输出要求
- 必须返回：
  - 提交哈希
  - 提交标题
  - 提交正文中的变更摘要
  - 本次提交文件列表
  - 合并来源分支与 main 合并结果（成功/跳过/失败）
  - main 当前最新提交哈希
  - `origin` 推送结果（成功/失败）
  - `github` 推送结果（成功/失败）
  - 是否存在残留未提交改动

## 边界与安全
- 禁止执行破坏性命令（如 `git reset --hard`、`git checkout --`）。
- 未经用户明确要求，不执行历史改写（如 `--amend`、`rebase`）。
- 未经用户明确要求，不新增或改写远程配置（包括 `origin pushurl` 与 `github` 远程）。
- 未经用户明确要求，不对远程执行 `--force` 或 `--force-with-lease` 推送。
- 若提交失败，返回失败原因与下一步建议（如先解决冲突、先暂存文件、先修复 pre-commit 报错）。
