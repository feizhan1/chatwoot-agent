# Git 中文提交技能

## 输入前提
- 运行目录必须位于 Git 仓库内。
- 允许读取 `git status` 与 `git diff` 结果生成提交说明。
- 若用户明确指定提交范围或提交信息，优先遵循用户要求。

## 执行步骤
1. 检查提交前状态。
- 运行 `git status --short`。
- 若无改动，直接返回“当前无可提交内容”，不执行 commit。

2. 确定提交范围。
- 若用户给出文件/目录路径，仅暂存这些路径。
- 若用户未指定范围：
  - 若已有 staged 内容，保持现状。
  - 若没有 staged 内容，执行 `git add -A`。

3. 生成中文提交说明。
- 提交标题格式：`<type>: <中文总结>`。
- `type` 优先使用 `feat`、`fix`、`docs`、`refactor`、`chore` 中最贴切的一类。
- 中文总结要求：
  - 8-30 字，聚焦“本次改动做了什么”。
  - 禁止纯英文整句。
  - 禁止空泛描述（如“更新一下”“改了点东西”）。

4. 执行提交。
- 运行 `git commit -m "<type>: <中文总结>"`。
- 默认不使用 `--no-verify`，除非用户明确要求跳过钩子。

5. 提交后校验。
- 运行 `git show --name-only --pretty=format:"%H%n%s" -1` 输出提交哈希与标题。
- 再次运行 `git status --short`。
- 若钩子产生新的未提交改动，明确告知用户并询问是否继续追加提交。

## 输出要求
- 必须返回：
  - 提交哈希
  - 提交标题
  - 本次提交文件列表
  - 是否存在残留未提交改动

## 边界与安全
- 禁止执行破坏性命令（如 `git reset --hard`、`git checkout --`）。
- 未经用户明确要求，不执行历史改写（如 `--amend`、`rebase`）。
- 若提交失败，返回失败原因与下一步建议（如先解决冲突、先暂存文件、先修复 pre-commit 报错）。
